Part 2 · 트랜스포머의 한계

Self-Attention의 근본 문제

모든 단어가 서로를 보는 대가 — 단어가 늘수록 연결(연산)이 제곱으로 폭발

어텐션의 비용 문제

  • 원인 — 어텐션은 모든 단어 쌍을 비교함. 단어가 L개면 연결은 L×L, 즉 길이의 제곱(L²).
  • 얼마나 심한가 — 길이 2배면 비용 4배, 10배면 100배. 앞 장의 KV 캐시(메모리)도 함께 늘어남.
  • 특히 치명적인 곳 — 100만+ 단어 문서, 고해상도 이미지, 긴 영상·오디오처럼 입력이 클 때.
  • 핵심 — 이건 어텐션이 태생적으로 지닌 비용임. 더 똑똑한 어텐션으로는 못 풂.

연산량 곡선 (길이 $L$ 대비)

Part 3 · SSM 소개

SSM — 메모리 하나로 처리하는 선형 방식

State Space Model — 어텐션과 정반대로, 순서대로 읽으며 '요약 메모리' 하나만 갱신

SSM이란

SSM(상태공간모델)은 단어를 순서대로 읽으며, 지금까지의 내용을 '메모리(상태)' 하나에 요약해 계속 갱신하는 방식임. 어텐션이 매번 전체를 다시 보는 것과 정반대죠. (RNN도 같은 계열)
  • 장점 — 메모리 크기가 고정이라 문장이 길어져도 비용이 일정(선형). 어텐션의 KV 캐시 폭발이 없음.
  • 한계 — 모든 단어를 똑같은 규칙으로 처리함('the'든 'cat'이든). 그래서 '중요한 걸 골라 기억'을 못 함.

Part 3 · ★ Mamba 소개

Mamba-1: 선택적 상태공간 모델(Selective SSM)

입력 내용에 따라 "무엇을 기억/망각할지" 스스로 정하는 가변 게이트

단 하나의 아이디어: "규칙을 입력에 따라 바꾼다"

고전 SSM은 모든 단어를 똑같은 규칙으로 처리했음. Mamba는 "지금 이 단어가 중요한가?"를 보고 규칙(게이트)을 매 순간 바꿈.
  • 중요한 단어("cat", "sat") → 게이트를 활짝 열어 메모리에 강하게 기록.
  • 안 중요한 단어("the", "on") → 게이트를 닫아 그냥 흘려보내고 기존 메모리 유지.
  • 즉, 내용에 따라 "무엇을 기억하고 무엇을 버릴지" 스스로 선택 — 고전 SSM이 못 하던 바로 그 일임.
  • 대가: 규칙이 매번 바뀌니 계산이 까다로워짐. 이건 GPU 메모리를 영리하게 쓰는 방식으로 해결함.

Part 3 · Mamba 설명 · 구현

Mamba — 메모리 최적화

규칙이 매번 바뀌어 계산이 까다롭지만, GPU의 '빠른 메모리'를 영리하게 써서 극복

핵심 아이디어

GPU에는 느리지만 큰 메모리와 빠르지만 작은 메모리가 있음. Mamba는 계산을 빠른 메모리 안에서 다 끝내고 결과만 큰 메모리로 내보냄. 느린 메모리를 왔다 갔다 하는 시간을 없애는 거죠.
  • 중간 계산을 저장하지 않고 필요할 때 다시 계산해 메모리를 아낌.
  • 순서대로 하던 계산을 병렬로 재조직해 GPU를 꽉 씀.
결과(원논문 기준): 이 방식으로 핵심 계산이 일반 구현보다 약 40배 빠르고, 글 생성 속도는 트랜스포머 대비 약 5배임.

느린 메모리 ↔ 빠른 메모리

추론 메모리: O(1) vs O(L)

  • 추론 메모리: Transformer는 토큰마다 K/V 누적 → 선형 증가. Mamba는 고정 크기 상태만 유지 → 상수.
  • 속도(원논문): 생성 추론 처리량 Transformer 대비 ~5×, 학습 커널은 naive scan 대비 ~40× — 서로 기준이 다른 별개 지표.
  • 한계: 특정 단어를 콕 집어 다시 찾아오는 능력은 순수 Mamba가 약함.
Part 3 · 둘의 공통점

Mamba의 약점, 그리고 하이브리드

정확히 찾아오기가 약한 것이 Mamba의 단점 — 어텐션을 소수 섞어 보완

단점 → 해법

  • 단점 — Mamba는 메모리에 요약만 하므로, 특정 단어를 콕 집어 다시 찾아오는 능력이 약함.
  • 해법 — 어텐션 층을 소수 섞음(하이브리드).
  • 섞을 수 있는 이유 — SSM과 어텐션은 결국 "어떤 단어에 얼마나 주목할지"를 계산하는 같은 일. 방법만 다를 뿐(어텐션=한 번에, SSM=순서대로 요약)이라, 한 모델에 섞어도 자연스럽게 이어짐.

두 방법, 같은 결과

두 길이 같은 곳에 도착하니, 한 모델 안에 번갈아 넣어도 자연스럽게 이어짐.
Part 3 · 하이브리드

하이브리드 구조

대부분 Mamba(효율) + 가끔 어텐션(정확히 찾기), 일정 간격 교차

실제 사례

기본 아이디어 — 층의 대부분은 Mamba(빠르고 메모리 적음), 일정 간격마다 어텐션 층 하나. Mamba가 약한 '정확히 찾아오기'를 그 어텐션 층이 담당.
  • Jamba (언어) — Mamba 층 사이사이에 어텐션 교차. 매우 긴 문맥(256K)을 효율적으로 처리.
  • MambaVision (이미지) — 뒷부분 층에만 어텐션 배치, 정확도와 속도를 함께.
  • 교훈 — "트랜스포머의 끝"이 아니라 둘의 역할 분담.

Part 3 · 도메인 확장

이미지 처리

Mamba는 '한 줄로 읽는' 방식 — 2차원 이미지는 '읽는 순서'를 새로 설계

문제와 해법 — 이미지를 어떻게 읽나

문제 — 이미지는 가로·세로 2차원임. 글처럼 한 줄로 쭉 펴면, 위아래로 붙어 있던 이웃 픽셀이 멀어져 관계가 끊김.
해법 — 한 방향이 아니라 여러 방향(가로·세로·역방향)으로 스캔함. 그러면 모든 픽셀이 사방 이웃의 정보를 받음. (위 격자가 그 4방향 스캔)

적용 분야

  • 의료영상 (CT·MRI) — 해상도가 매우 커서 어텐션의 제곱 비용이 치명적. Mamba의 선형 비용이 특히 유리.
  • 긴 영상·3D 점데이터 — 프레임이나 3차원 점도 '읽는 순서'만 잘 정하면 Mamba로 처리 가능.
  • 공통 교훈 — 순서 없는 데이터에선 '어떤 순서로 읽을지' 설계가 성능을 좌우.
Conclusion

Mamba의 현재 한계

강력한 새 축이지만, 아직 트랜스포머를 완전히 대체하진 못함

주요 한계

  • 정확히 찾아오기 약함 — 특정 단어를 콕 집어 회상하는 능력이 어텐션보다 부족.
  • 읽는 순서 미표준 — 이미지·3D처럼 순서 없는 데이터의 범용 스캔 방법 부재.
  • 초대형 검증 부족 — 아주 큰 모델에서 트랜스포머만큼 안정적인지 검증 필요.

그럼에도 — 강한 영역

  • 긴 입력 — 아주 긴 글·영상, 고해상도 이미지처럼 입력이 클수록 강점.
  • 상수 메모리 — 문맥이 길어져도 메모리가 늘지 않아 저지연·저비용.
  • 현실적 방향 — 완전 대체가 아니라, 어텐션과 섞는 하이브리드로 각자의 강점 활용.

💡 IDEATION · 보너스 트랙

생성 방식 — 자기회귀 vs 확산

자기회귀는 단어를 하나씩, 확산은 문장 전체를 동시에 정정

① 자기회귀 (지금까지의 방식)

단어를 왼쪽 → 오른쪽으로 하나씩 만듦. 한 번 뱉은 단어는 다시 못 고침. 그리고 한 번에 하나씩이라 단일 사용자에겐 느림.

② 확산 (Diffusion) 생성

문장 전체를 흐릿하게(빈칸으로) 깔아놓고, 여러 번 반복하며 동시에 또렷하게 채움. → 병렬이라 빠르고, 틀린 단어를 다시 고칠 수 있음(자기수정).
💡 IDEATION · 내 아이디어

확산 × Mamba (MD³)

Jamba가 Transformer+Mamba를 섞었듯, 확산 생성의 '속(백본)'을 Mamba로 바꾸면?

문제 → 해법

  • 확산의 장점 — 병렬 생성(빠름) + 자기수정(품질).
  • 그런데 문제 — 지금 확산 모델도 속이 Transformer라, 우리가 본 O(L²) + 메모리 증가가 또 나옴.
  • 해법(MD³) — 속을 Mamba로 교체. 병렬 생성은 그대로 두고, 비용은 선형 + 메모리 상수로.
정직하게 — 이런 방향(DiffuApriel, 2025)이 이미 막 시작됐음. 내 MD³의 차별점 = ① 장문에서도 메모리 상수(상태 캐시), ② Mamba-3 회전을 앞뒤 양방향으로.

문제 해결 의도

  • 왜 확산? 병렬·자기수정 → 단일 사용자 지연을 낮춤.
  • 왜 Mamba? 확산의 두 비용(O(L²)·메모리 증가)을 정확히 없애는 게 선형·고정메모리 SSM.
  • 왜 양방향? 확산은 '빈칸 채우기'라 앞뒤를 다 봐야 함 → Mamba를 정/역 두 방향으로 굴림.
  • 왜 어텐션을 조금? Mamba 혼자선 '정확히 찾아오기'(검색)가 약해 소수 어텐션 층으로 보강.

장점 vs 정직한 한계

효율은 좋아지지만, 품질은 별도 보강 필요

기대 장점

  • 저지연 — 병렬 생성으로 단일 사용자 응답이 빠름.
  • 메모리 상수 — 아무리 긴 글을 써도 메모리가 일정.
  • 자기수정 — 틀린 단어를 다시 마스킹·정정.
  • 상태추적 — Mamba-3 회전이 규칙적 패턴(짝/홀 등)에 강함.

정직한 한계

  • 효율만 확실히 상보적 — 품질·검색은 자동으로 안 됨 → 소수 어텐션 + 자기수정으로 근거 기반 보강.
  • 아직 설계·이론 단계 — 실험 검증은 앞으로.
  • 헤드라인은 선점(확산×Mamba) — 내 델타는 상수 메모리·회전 양방향.
