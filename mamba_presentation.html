Part 01-A

Transformer 아키텍처와 Self-Attention 기술 원리

자가주의 메커니즘을 통한 전역 문맥 인코딩과 계산 병렬화

Self-Attention 기술 정의 및 수학적 메커니즘

트랜스포머는 기존 순차 구조(RNN)를 탈피하고, 시퀀스 내 모든 토큰 쌍의 상호작용을 한 번에 계산하는 Self-Attention(자가주의) 연산을 핵심으로 합니다.

  • Q, K, V 사상: 각 토큰 임베딩에 가중치 행렬을 곱해 Query·Key·Value 세 벡터를 만듭니다. "무엇을 찾는가(Q)·무엇을 가졌는가(K)·실제 내용(V)".
  • 스케일 내적 어텐션: $QK^T$로 토큰 간 유사도를 구하고 $\sqrt{d_k}$로 나눠 안정화한 뒤 Softmax로 가중치를 만들어 $V$와 곱합니다.
  • 병렬성과 전역성: RNN과 달리 전 토큰을 동시에 처리(학습 병렬화)하고, 거리에 상관없이 직접 연결합니다. 단, 그 대가가 $O(L^2)$ 비용입니다(뒤에서 다룸).
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

GPT-3 (2020) 아키텍처 해부 — Dense Transformer의 정점

96개 동일 디코더 블록의 단순 적층 · 이후 모든 비교의 기준점

설계 스펙 (당시 표준)

GPT-3 175B는 동일한 디코더 블록 96개를 단순 적층한 Dense 모델입니다. 2026년 관점에서는 "고전 구성"의 교과서이며, 이후 슬라이드의 모든 진화가 이 기준선을 개선합니다.

  • 파라미터: 175B (전부 활성 · Dense)
  • 레이어 / 폭: 96 layers, $d_{model}=12288$, 96 heads
  • 어텐션: Multi-Head Attention (MHA) — Q/K/V 헤드 96개 동수
  • 위치 인코딩: Learned Absolute Position Embedding
  • 정규화: Pre-LayerNorm (서브블록 앞, GPT-2 계승 — 원조 2017 Transformer의 Post-LN과 다름)
  • 활성함수: GeLU · 컨텍스트: 2,048 · 어휘: 50,257
→ 4가지(LayerNorm · Absolute PE · MHA · GeLU)가 이후 6년간 각각 RMSNorm · RoPE · GQA · SwiGLU로 교체되고, Dense는 MoE로 바뀝니다.

Part 02-B

위치 인코딩 혁명: Learned Absolute → RoPE

위치를 "더하기" 대신 Query·Key를 각도로 "회전"시켜 상대 거리를 주입

RoPE 원리

🧭 한 줄 직관 — 토큰마다 위치를 시곗바늘 각도로 표시한다고 생각하세요. 멀리 떨어진 토큰일수록 바늘이 더 많이 돌아가 있고, 두 바늘의 각도 차이가 곧 ‘서로 얼마나 떨어졌는가’가 됩니다.
  • 기존 방식의 한계: GPT-3는 위치마다 학습된 벡터를 토큰 임베딩에 더했습니다(Learned Absolute). 그런데 학습 때 본 길이를 넘어가면 그 위치 벡터가 없어 외삽이 불가 → 컨텍스트 확장이 어렵습니다.
  • RoPE의 발상 (3단계): ① Query·Key 벡터를 2차원씩 묶어 ② 위치 $m$에 비례한 각도 $m\theta$만큼 회전시키고 ③ 어텐션 내적을 계산합니다. 그러면 결과가 두 위치의 차 $(m-n)$에만 의존 → 절대 위치가 아니라 상대 거리가 자연히 들어갑니다.
  • 왜 더 좋은가: 길이에 무관한 상대 패턴을 학습하므로 학습 길이 너머로도 일반화가 잘 되고, 위치를 임베딩에 더하지 않아 어텐션 단계에서 깔끔하게 적용됩니다.
  • 2026 확장: Qwen3는 RoPE로 최대 256K. Llama 4는 iRoPE — RoPE 레이어와 위치인코딩 없는(NoPE) 전역 어텐션 레이어를 교차 배치해 최대 10M(Scout)을 노림.
$$\langle R_m q,\, R_n k\rangle = \langle q,\, R_{n-m}k\rangle \quad\text{(내적이 } n-m \text{에만 의존)}$$

어텐션 진화: MHA → MQA → GQA

KV Cache 메모리 대역폭 병목을 줄이기 위한 헤드 공유 전략

MHA (GPT-3)

헤드마다 독립 K/V → 품질 최고, 메모리 최대.

GQA (2026 표준)

Part 02-D

활성함수 진화: GeLU → SwiGLU (게이트형 FFN)

단일 비선형 함수에서, 곱셈 게이트로 정보 흐름을 제어하는 구조로

SwiGLU 게이트 메커니즘

GPT-3의 FFN은 $\text{GeLU}(xW_1)W_2$로 단일 비선형이었습니다. SwiGLU는 입력을 두 경로로 갈라, 하나를 게이트로 써서 다른 경로를 원소곱으로 변조합니다.

  • Swish/SiLU 게이트: $\text{SiLU}(x)=x\cdot\sigma(x)$가 "얼마나 통과시킬지"를 부드럽게 결정.
  • 표현력 ↑: 동일 파라미터에서 게이트 곱셈이 더 풍부한 함수를 학습 → 2026 모델 거의 전부 채택.
  • Mamba 연관: Mamba 블록도 SiLU 게이트(원소곱 $z$ 경로)를 사용 — "게이트형 구조"는 SSM과 Transformer가 공유하는 설계.
$$\text{SwiGLU}(x)=\big(\text{SiLU}(xW_g)\odot xW_u\big)W_d$$

Dense → Mixture of Experts (MoE)

파라미터는 키우되 토큰당 연산은 일정하게 — "활성 파라미터" 시대

희소 활성화의 핵심

  • 라우터(Gating)가 각 토큰에 대해 상위 $k$개의 전문가(FFN)만 선택 → 전체 파라미터의 일부만 활성.
  • 총 vs 활성: Qwen3-235B는 22B만 활성(A22B), Llama 4 Maverick은 400B 중 ~17B 활성.

  • Mamba와의 결합: BlackMamba·Jamba처럼 SSM mixer + MoE FFN 조합도 등장.

Part 04-A

현대 LLM의 추론 방식: Decoder-Only 자기회귀

2017 원조 Transformer는 인코더–디코더였지만, GPT·Qwen·Llama·Gemini는 모두 디코더 전용

인코더–디코더 → 디코더 전용

  • 2017 원조 Transformer: 번역용 인코더–디코더 구조 (입력을 인코더가 요약 → 디코더가 Cross-Attention으로 참조하며 출력).
  • 2020~2026 주류 LLM: GPT-3·Qwen3·Llama 4·Gemini 3 모두 디코더 전용(Decoder-Only). 인코더·Cross-Attention 없이, 한 스택이 다음 토큰만 반복 예측.
  • 자기회귀(Autoregressive): 이미 생성한 토큰을 다시 입력에 붙여 그 다음 토큰을 예측 — 한 번에 한 토큰씩.
  • Causal Masking: 각 토큰은 자신과 과거만 어텐션 (미래 차단). 그래서 번역·요약·대화·코딩을 하나의 생성 문제로 통일.
→ 그래서 다음 슬라이드의 KV Cache 병목과 $O(L^2)$ 문제가 생성 단계마다 직접적으로 나타납니다.

Part 04-C · 한계 ②

$O(L^2)$ 한계와 선형 SSM의 동기

시퀀스 길이가 커질수록 벌어지는 연산량 격차 — SSM이 풀려는 문제

왜 선형이 필요한가

  • Transformer: 모든 토큰 쌍을 비교 → 연산·메모리 $O(L^2)$. $L$이 2배면 비용은 4배.
  • SSM / Mamba: 고정 크기 상태 $h_t$를 순차 갱신 → 학습 $O(L)$, 추론 토큰당 $O(1)$ (KV Cache 없음).
  • 임계 영역: 100만+ 토큰 문서, 고해상도 이미지(수만 패치), 3D 의료영상, 장시간 비디오·오디오에서 격차가 폭발적.
  • 결론: "더 똑똑한 어텐션"이 아니라 복잡도 차원을 바꾸는 접근 — 이것이 Mamba의 출발점.

RNN & SSM의 메모리 저장 방식

RNN계열과 초기 SSM(S4, H3 등)은 전체 입력을 다 들여다보는 대신, 매 시점 업데이트되는 고정된 크기($N$)의 숨겨진 상태 벡터 $h_t$만을 기억합니다.

  • 추론 장점: KV cache 가 없어 메모리 증가 없음 ($O(1)$)
  • 훈련 장점 (LTI SSM): 전이 파라미터가 고정되어 있으면 전체 수식을 합성곱(Convolution)으로 변환하여 FFT로 고속 병렬 학습 가능
  • 핵심 한계: 선형 시불변(LTI) 모델은 입력 내용에 따라 "무엇을 기억하고 무엇을 버릴지" 능동적으로 선택할 수 없음

Part 05-B · SSM 수학

SSM 기초: 연속 상태공간 → 이산 recurrence

미분방정식을 Zero-Order Hold로 이산화 → RNN형 점화식 / 합성곱 이중 표현

연속 → 이산 변환

고전 SSM은 연속시간 선형 시스템에서 출발합니다. 입력 $x(t)$를 받아 잠재 상태 $h(t)$를 통해 출력 $y(t)$를 냅니다.

$$h'(t)=Ah(t)+Bx(t),\quad y(t)=Ch(t)$$

스텝 사이즈 $\Delta$로 Zero-Order Hold 이산화하면 RNN형 점화식이 됩니다.

$$\bar A=e^{\Delta A},\;\bar B=(\Delta A)^{-1}(e^{\Delta A}-I)\Delta B$$ $$h_t=\bar A h_{t-1}+\bar B x_t,\quad y_t=Ch_t$$
핵심 트레이드오프: 파라미터가 고정(LTI)이라 두 표현(점화식↔합성곱)을 자유롭게 오가며 FFT로 병렬 학습할 수 있습니다. 바로 이 "고정성"을 깨는 것이 다음 슬라이드 Mamba의 출발점입니다.

이중 표현: Recurrent ⇄ Convolution

Part 05-C · ★ Mamba 등장

Mamba-1 등장: 선택적 상태공간 모델(Selective SSM)

한 줄 핵심 — 입력 내용에 따라 "무엇을 기억/망각할지" 스스로 정하는 가변 게이트

가변적 discretization 파라미터

Mamba는 전이 조건 파라미터 ($B_t$, $C_t$, $\Delta_t$)를 입력값 $x_t$의 실시간 함수로 설정합니다. 중요 키워드 유무에 따라 스텝 사이즈 $\Delta_t$가 변화하여, 메모리 저장 주기를 동적으로 재구성합니다.

  • 핵심 원리: 중요 단어("cat", "sat") 등장 시 $\Delta_t$가 증가하여 이전 잡음 상태를 망각하고 현재 정보를 강하게 기입
  • 하드웨어 최적화: 합성곱 병렬화 불가 문제를 극복하기 위해, 느린 HBM을 건너뛰고 GPU 온칩 SRAM 내부에서 discretize와 scan을 원스톱으로 처리하는 Fused Selective Scan Kernel 설계

Part 05-D · 블록 해부

Mamba 블록 해부 — 전체 데이터 흐름

입력 투영 → Conv1D → 선택적 SSM(Δ,B,C는 입력의 함수) → SiLU 게이트 → 출력 투영

핵심 차별점: 기존 SSM은 $B,C,\Delta$가 고정(LTI) → Mamba는 이들을 입력 $x_t$의 함수로 만들어 "무엇을 기억/망각할지" 내용 기반 선택. 그 대가로 합성곱이 불가 → Hardware-Aware selective scan으로 해결(다음 슬라이드).
Part 05-E · 구현

Hardware-Aware Selective Scan

선택성 때문에 합성곱이 불가 → GPU 메모리 계층을 의식한 fused kernel로 극복

세 가지 핵심 기법

  • Kernel Fusion: 상태 $h$ 전체를 느린 HBM에 물질화하지 않음. 파라미터를 SRAM으로 올려 그 안에서 이산화+scan을 끝내고 출력만 HBM에 기록.
  • Parallel Scan: 순차 점화식을 결합법칙을 이용한 병렬 prefix-scan으로 변환 → GPU 병렬성 활용.
  • Recomputation: 역전파 시 거대한 중간 상태를 저장하지 않고 재계산 → 메모리 절감.
결과(원논문, A100): 선택적 스캔 커널이 표준(naive) 구현 대비 ~40× 빠름, 생성 추론 처리량은 Transformer 대비 ~5×. FlashAttention과 유사한 메모리 프로파일.

추론 메모리: O(1) vs O(L)

  • 추론 메모리: Transformer는 토큰마다 K/V 누적 → 선형 증가. Mamba는 고정 크기 상태만 유지 → 상수.
  • 속도(원논문): 생성 추론 처리량 Transformer 대비 ~5×, 학습 커널은 naive scan 대비 ~40× — 서로 기준이 다른 별개 지표.
  • 한계: 직접적 content retrieval(연상 회상)은 순수 Mamba가 약함 → 하이브리드로 보완.
Part 06-A · 진화

Mamba-2: State Space Duality (SSD)

SSM 순차 전이와 어텐션 행렬의 이론적 연결 고리

Semiseparable Matrix로 정의한 쌍대 관계

Mamba-2는 1-준분해 가능 행렬(Semiseparable Matrix) 표현식을 통해 **SSM의 상태 갱신이 곧 특수한 형태의 마스크 어텐션과 연산 구조적으로 동일하다**는 점을 증명했습니다.

  • Attention 관점: Gram matrix $QK^\top$에 감쇠 상태 전이 마스크 $L$을 융합한 형태
  • 블록 분해 기법: 시퀀스를 블록으로 분할하여, 블록 내부는 GPU 텐서 코어에 친화적인 행렬 곱(Attention)으로 연산하고 블록 사이는 SSM 상태 전이로 처리
  • 구조 단순화: $X, B, C, \Delta$를 일괄적으로 전사하는 단일 평행 프로젝션(GQA 유사)으로 아키텍처 다이어트

이론 수식 및 계산 그래프

$$Y_i = C_i \sum_{j \le i} \left( \prod_{k=j+1}^i A_k \right) B_j X_j$$

Part 06-B · 생태계

하이브리드 아키텍처: Jamba · MambaVision

순수 Mamba의 retrieval 약점을 Attention 레이어를 교차 배치해 보완

왜 섞는가

  • Jamba (AI21, 언어): Transformer·Mamba 레이어를 교차 배치 + MoE. 효율(Mamba)과 retrieval 품질(Attention)을 동시 확보, 256K 장문맥. 1.5 Mini는 12B·Large는 94B 활성 파라미터(MoE).
  • MambaVision (NVIDIA, 비전): 후반부 레이어에만 self-attention을 재투입 → ImageNet/COCO에서 강한 정확도·처리량 절충 (CVPR 2025).
  • BlackMamba: Mamba mixer + MoE FFN으로 희소 용량 확장.
  • 교훈: 2024–26 흐름은 "Transformer 종식"이 아니라 SSM과 Attention의 역할 분담.
